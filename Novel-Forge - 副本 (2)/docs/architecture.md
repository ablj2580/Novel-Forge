# 架构草案

## 界面层

### 主视图结构
- **书库面板**：左侧作品列表，支持多作品切换，颜色区分书目
- **模块导航**：设定资料库的模块列表，支持独立滚动
- **内容区域**：根据当前视图展示对应内容

### 视图模块
1. **创意整理台**：灵感收件箱，快速记录和 AI 审阅
2. **设定资料库**：展示各模块的设定资料，支持搜索和筛选
3. **关系图谱**：可视化展示设定资料之间的关系
4. **正文写作**：章节编辑器和右侧相关设定引用

### 交互特性
- **搜索系统**：全局搜索设定资料，支持标题、内容、标签匹配
- **主角标记**：每本书可设置唯一主角，关系图谱默认以主角为中心
- **编辑模式**：灵感收件箱支持编辑标题、内容、模块、标签、归属作品

## 核心逻辑层

### AI 分类引擎
- 识别作品归属、分类、提取标题/摘要/标签/置信度
- 支持自定义模块识别，动态生成提示词

### 搜索索引
- Demo 使用前端全文匹配（标题、内容、标签）
- 按当前作品过滤搜索结果
- SQLite 版建议使用 FTS5

### 关系管理
- **关系类型管理**：支持新增、编辑、删除，自动使用最小可用ID
- **关联建立**：搜索设定资料建立关系，按当前作品过滤

### API 代理
- Demo 已通过 `/api/chat-completion` 代理 DeepSeek 请求
- 支持配置不同 AI 提供商

### 导出模块
- 当前支持 JSON 备份
- 后续可扩展：Markdown、DOCX、PDF

## 数据层

### 当前实现（Demo）
- 使用浏览器 `localStorage` 存储
- 数据对象按未来 SQLite 表结构设计

### 数据模型
- **BOOKS**：作品信息
- **NOTES_RAW**：原始灵感笔记
- **IDEAS**：设定资料（设定资料库条目）
- **CHARACTERS**：人物专项卡片
- **WORLDBUILDING**：世界观专项卡片
- **CHAPTERS**：章节内容
- **RELATIONS**：设定资料之间的关系
- **RELATION_TYPES**：关系类型定义
- **AI_CONFIG**：AI 接口配置

### 正式版规划（Windows 桌面版）
- SQLite 主库 + FTS5 搜索索引 + 本地文件目录
- API Key 使用系统凭据库或本地加密存储

## 状态管理

### 全局状态
- `activeView`：当前视图（inbox/library/relations/writing）
- `selectedBookId`：当前选中的作品
- `selectedModuleId`：当前选中的设定模块
- `selectedNoteId`：当前选中的灵感笔记
- `selectedItemId`：当前选中的设定资料

### 状态同步
- 作品切换时同步更新设定资料库和关系图谱
- 编辑操作自动保存并重新渲染相关视图

## 关键函数

### 渲染函数
- `render()`：主渲染入口
- `renderNoteDetail()`：渲染灵感笔记详情
- `renderLibrary()`：渲染设定资料库
- `renderRelationGraph()`：渲染关系图谱
- `renderSearchSuggestions()`：渲染搜索建议

### 业务函数
- `acceptSuggestion()`：确认归档灵感到设定资料库
- `editNote()`：编辑灵感笔记
- `addRelation()`：添加设定资料关系
- `rollbackLibraryItem()`：回退设定资料到收件箱
